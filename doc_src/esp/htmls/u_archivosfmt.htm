Archivos de formato para DOSPrint
Actualizado al: 07-OCT-2001

Un archivo de formato para DOSPrint es un archivo de texto que contiene la definición de un informe DOSPrint. Este archivo de texto contiene la definición de cada una de las secciones de un informe, así como la configuración de página a utilizar, los grupos de control existentes y los macros a utilizar, el cual luego es utilizado por el método Run de un objeto DOSPrint

 

Un archivo de formato está compuesto por secciones, algunas de las cuales se consideran "fijas". Cada sección inicia con el tag <SECCION> y termina con el tag </SECCION>. Adicionalmente, toda linea que comienze con el caracter # es considerada comentario. Tambien se considera comentario todo texto que siga a los caracteres //.

 

Las secciones fijas de un archivo de formato son:

 

  • Sección CONFIG
  • Sección MACROS
  • Sección GROUPS
  • Sección TITLE
  • Sección HEADER
  • Sección DETAIL
  • Sección FOOTER
  • Sección SUMMARY

 

Adicionalmente a estas secciones, se puede crear cualquier sección deseada, simplemente usando los tags de inicio y cierre, por ejemplo:

 

<ESPECIAL1>
Esto es una sección especial dentro de un archivo de formato. El contenido
de esta sección puede ser impresa a través del método PrintSection() ($DATE$).
</ESPECIAL1>

Una vez definida la sección en el formato, se utiliza el método PrintSection para imprimir dicha sección.

Ejemplo:

 

# INFBASCLI.FMT
# Listado básico de clientes
#
<config>
StartConfString=$C10$$COFF$    // 10 CPI sin compresión (80 cols)
PaperLenght=60
TopMargin=2
LeftMargin=5                   // Margen para encuadernación
</config>

<macros>
CSTATUS=proper(STATUS)
EMPRESA='Tuberias XYZ'
DIRECCIONEMP='Caracas, Venezuela'
</macros>

<header>
#...+....[....+....[....+....1....+....1....+....1....+....1....+....1....+....1
[EMPRESA                          ]                   LISTADO BASICO DE CLIENTES
[DIRECCIONEMP                     ]

CODIGO                NOMBRE                                 STATUS
====================  ====================================== ==========
</header>

<detail>
[CODIGO            ]  [PROPER(NOMBRE)                      ] [CSTATUS]
</detail>


<footer>


[datetime           ]                                                   [pageno]
</footer>
