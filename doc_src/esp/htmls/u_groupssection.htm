Sección GROUPS

La sección GROUPS define los diferentes grupos de control que manejará el informe. Cada linea en la sección define un grupo de control mediante la siguiente sintáxis:

 

GROUP=Expr | Header | Footer

 

donde:

Expr Expresión que controlará el salto de grupo. Cuando esta expresión cambie, se producirá una ruptura de control en el grupo.
Header Nombre de una sección personalizada que servirá como encabezado del grupo.
Footer Nombre de una sección personalziada que servirá como pie de grupo.

 

 

Si el informe maneja varios grupos de control, los mismos serán evaluado en el orden en que son definidos en la sección.

Ejemplo:

 

# INFBASCLI.FMT
# Listado básico de clientes, agrupado por status.
#
<config>
StartConfString=$C10$$COFF$    // 10 CPI sin compresión (80 cols)
PaperLenght=60
TopMargin=2
LeftMargin=5                   // Margen para encuadernación
EjectBeforeSummary=No
</config>

<macros>
CSTATUS=proper(STATUS)
EMPRESA='Tuberias XYZ'
DIRECCIONEMP='Caracas, Venezuela'
TNUMCLI=TRANS(_COUNT[],"9,999")          // Nro. de clientes total
GNUMCLI=TRANS(_COUNT[STATUS],"9,999")    // Nro. de clientes por status
</macros>

<groups>
XSTATUS=STATUS | XTATUS_HDR | XSTATUS_FTR
</groups>

<header>
#...+....[....+....[....+....1....+....1....+....1....+....1....+....1....+....1
[EMPRESA                          ]                   LISTADO BASICO DE CLIENTES
[DIRECCIONEMP                     ]                                 SEGUN STATUS

CODIGO                NOMBRE                                 STATUS
====================  ====================================== ==========
</header>

<xstatus_hdr>

STATUS: $BON$%PROPER(STATUS)%$BOFF$
</xstatus_hdr>

<detail>
[CODIGO            ]  [PROPER(NOMBRE)                      ] [CSTATUS]
</detail>


<xstatus_ftr>

TOTAL CLIENTES: [GNUMCLI]

 

 

</xstatus_ftr>

<footer>


[datetime           ]                                                   [pageno]
</footer>

<summary>

************************
TOTAL CLIENTES: [NUMCLI]
************************
</summary>
