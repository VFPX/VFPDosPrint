What's new

New features and fixes in version 4.2:

Area Feature name Description
New properties PDEOF This new runtime variable can be used with ApplyIf property to define a section that will only be printed at the end of report.

The property evaluates to True after the last detail record is printed, wich makes it suitable to be used in Page or group footer sections.
 
Improvments Clear method The Clear method was redesigned to allow the creation of a public instance of DP rather than local instances that are destroyed after been used. With this version, you can create a single public instance of DP and then call the Clear method just before the printout process.

Fixed bugs:

Additionally, the following know bugs where fixed:

  • When a footer section does not fit in the current page, a page break is generated but the Header section is not printed.
  • Current value for defined macro went to blanks where macros are used in Summary section or Footer section in the last page of the report.
  • If an expandible expression delimited with [ ] was incorrect, the next expandible expression is not expanded either.

 

 

New predefined macro PageCnt:

Yes! now you can put a 'Page 0 of 0' in your DP reports, using the new predefined macro PageCnt. Just that easy: put a %PageCnt% inside your format file and the macro will be expanded at the end of the report.

 

 

VFP5 Compatibility:

This version of DP is no longer compatible with Visual FoxPro 5.0.
